Blazor is a fresh take on the well-estabished Razor/C# combo for building UI components. It pairs quite well with Tailwind CSS, which will significantly reduce the amount of custom CSS you have to write, keep styling local to your Razor components, while also allowing for things like themes, variables, and nesting. Rather than a set of static utility classes, it is effectively a CSS generator, building classes on demand based on an overridable and extendable base set of values and utilities. The documentation is great as well. (Random example: position.)

"Inner Loop" goodies

Blazor's Hot Reload allows edits to be applied immediately without needing to pause or restart a running app. This includes edits to code as well as markup and CSS. It's still newish and not fully supported for all project types or code edits.

Similar to .NET's Hot Reload, Tailwind's Just-In-Time CSS generation is the primary goodness Tailwind CSS 3 brings. Initially it generates only what CSS is relevant to your markup and CSS, subsequently performing much faster incremental builds when changes are spotted. Also newish, it works well until it doesn't - for me it gets "stuck" during a build and runs out of memory once every 2-3 hours.

Hot Reload and tailwindcss --watch do their work independently - what's needed from us is to point tailwindcss at our input markup/CSS, and our Blazor project at the tailwindcss output.

Tidy CSS

Blazor's take on CSS Isolation is a simple way to limit the scope of styles to a particular Razor Component: by convention, styles defined in .razor.css (Scoped CSS) files will be rewritten at build time so they only apply to their associated .razor Component.

In addition to generating classes for us, Tailwind CSS offers an @apply directive, and built-in support for Sass-like nesting is easily enabled.

To use Tailwind CSS features within your Scoped CSS files takes as little as three lines of code and is well worth it, IMO.

  • VS Code with some handy extensions:
    • The C# Extension.
